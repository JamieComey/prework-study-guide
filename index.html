HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • headers come in different sizes from 1 to 6 and should be used in that order from top to bottom for accessibility
  • Using theul elements creates an unordered list where each line must have a li element
  • We learned some basic CSS elements
  • We did some simple debugging to see why our CSS weren't applying and we learned how to fix that with the style element
  • We learned the three types (inline etc and we learned how to apply styles to multiple elements at once.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

CSS

    • A margin indicates how much space we want around the outside of an element.
    • A padding indicates how much space we want around the content inside an element.
    • we learned the different types of css (inline etc.)
    • we learned how to add the same style for multiple elements

Git

  • we learned a number of ccommands for git i.e git status (what branch) git checkout -b branch name (new branch and switch to it) etc.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • We learned about functions, being a sequence of events to run and we learned how to call them.
  • we used this and the console on the google chrome developer tools to check they worked and perform debugging.
